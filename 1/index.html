Lead Story

AI Agents Now Run the Homelab Around the Clock, and the Family Hasn't Noticed

Three months into the experiment, autonomous agents are no longer assisting with infrastructure tasks — they're running them. The household lives inside a system it doesn't think about.

By Alex Southwell · Infrastructure Desk · {{ "now" | date: "%B %-d, %Y" }}

Three months ago, the premise was simple: give AI coding agents access to a home network and see what happens. The answer, it turns out, is that they build things. They build dashboards and monitoring stacks, backup routines and health checks. They write configuration files with more care than most humans would bother with. And then they keep running.

Claude Code handles the audit layer — scanning Docker configurations for drift, reviewing Home Assistant automations for logical gaps, and proposing infrastructure changes through carefully structured pull requests. It reads configs like a careful site reliability engineer: methodically, skeptically, with an eye for what could go wrong at 3 AM.

"What's notable is not the individual components but how they compose. The recursion is the interesting part."

Codex operates differently. Where Claude Code is deliberate, Codex is fast. It translates natural-language requests into working containers, builds dashboards from descriptions, and ships features with the velocity of a startup developer on a deadline. The Kitchen Screen dashboard — now the household's primary information surface — was conceived and deployed in a single Codex session.

Then there's Clawdbot, the overnight operator. While the household sleeps, Clawdbot monitors container health, checks backup integrity, and watches for anomalies in the Home Assistant event stream. It escalates only when thresholds are breached, which happens rarely enough that its 3 AM notifications carry genuine weight.

The NurseDroid server remains the backbone: an Intel N100 mini-PC running eleven containers that serve Home Assistant, media streaming, backup management, and system monitoring. What began as a Raspberry Pi replacement has become a genuine home operations platform — stable, quiet, and increasingly self-managing.

The Kitchen Screen — a wall-mounted display running a custom agent-built dashboard — shows live sensor data, container health, and family scheduling. It's the household's control surface, updated by agents and consumed by humans who mostly don't think about what powers it. That invisibility is the point.

What's notable is not the individual components but how they compose. Each agent operates in a lane, each machine has a role, and the glue between them — configuration, monitoring, alerting — is itself maintained by agents. The infrastructure is becoming self-maintaining, which is either the dream or the beginning of a very specific kind of problem.

Analysis

The Agent Operating Model

Different agents suit different tasks. Claude Code excels at planning and auditing — it reads configs like a careful SRE. Codex is the sprint developer, shipping features fast. Clawdbot is the patient night watch, monitoring and reacting to events the household generates in its sleep.

The division of labor wasn't planned. It emerged from the constraints of each tool's strengths. That's probably the most interesting finding so far.

Essay

Infrastructure as Narrative

This site isn't documentation in the traditional sense. It's a running account of what happens when software agents interact with real household systems. The editorial format is deliberate: infrastructure deserves narrative attention.

Every Docker container has a story. Every Home Assistant automation was a decision. The broadsheet format makes those decisions visible.

Outlook

What's Next for the Homelab

Zigbee mesh expansion is the immediate priority — the current bridge is showing latency under load. Voice control integration via Home Assistant is in testing. And an experiment in letting agents propose their own infrastructure changes is being carefully scoped.

The homelab is becoming self-modifying. Carefully.

Opinion

The Agents Are Not the Point

Editorial Board

It's tempting to focus on the AI agents themselves — their capabilities, their limitations, their occasionally surprising decisions. But three months into this experiment, the more interesting observation is about the infrastructure they've built.

A household network that monitors itself. A dashboard that updates without human intervention. Backup routines that run because an agent decided they should. The agents are interesting, but the system they've composed is the actual artifact worth studying.

We've accidentally built a living system. The agents are the gardeners. The garden is the thing.
