Growing Season

A chronological view of how the garden took shape.

Week 1
Planting the seed
NurseDroid set up as the primary server. Docker installed. First containers: Home Assistant, Portainer, Mosquitto.
Week 3
First shoots
Kitchen Screen dashboard deployed by Codex in a single session. Home Assistant entity count passes 100. Zigbee mesh established.
Week 5
Agents take root
Clawdbot starts overnight monitoring. Claude Code begins auditing Docker configurations. The homelab starts to self-manage.
Week 8
The garden grows
Orange Pi RV2 added for edge experiments. Termux on Pixel becomes a mobile operations terminal. 264 HA entities.
Now
A mature system
Five machines, three agents, 11 containers. The family lives inside the infrastructure without thinking about it. Which is the whole point.

Why This Approach

Most homelab blogs treat infrastructure as a puzzle to solve — optimize, benchmark, stack-rank. This site treats it more like gardening.

You plant things. You water them. Some die. Some thrive in ways you didn't expect. The agents are gardeners with different temperaments, and the infrastructure is what grows.

What's Growing Next

Zigbee mesh expansion — the current bridge shows latency under load. Voice control integration is in testing. And an experiment in letting agents propose their own changes is being carefully scoped.

Slow and steady. Like a garden.
